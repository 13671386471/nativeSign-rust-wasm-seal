## 1. 修复 ToUnicode 去重导致 Kangxi 字根泄漏 (font_embed.rs)
- 问题: 嵌入产物文本提取出现 ⼀/⼄/⼯ (Kangxi, U+2F00~) 而非 一/乙/工。
- 根因: `build_to_unicode_cmap` 旧去重按 unicode 最小保留, 但 U+2F00(Kangxi) < U+4E00(统一汉字)
  且两者同 CID, 错误保留字根。
- 修复: 新增 `unicode_priority()`, 排序时按标准度优先再 dedup; 重建 pkg_node + pkg(web, release)。
- 验证(node 真 wasm + pypdf): Kangxi/兼容区/字根补充=0, 原文/嵌入 389 去重 CJK 完全一致, tofu=0。

## 2. 修复刷新页面 Aborted(FPDF_GetSystemFontInfo) (index.html)

## 5. 修复 preprocess_pdf_for_cjk 损坏 PDF 导致签章解析失败

### 问题
- 阳光电源 PDF 加载后签章数始终为 0，右键菜单无反应
- 日志: `[parse_seals] PDF加载失败: Invalid file trailer`
- 根因: preprocess_pdf_for_cjk 将 107KB PDF 处理为 8.9MB, 输出的 PDF 文件尾部损坏

### 修复
1. `parse_existing_signatures` 改为静态方法，接收任意 PDF 数据
2. `lib.rs::load_file()`: 先在原始数据上解析签章 → 预处理 → 加载 → 恢复签章统计
3. `engine.rs::load_file()` 移除内部的 parse_existing_signatures 调用

## 6. 增强签章属性字段提取（2026-07-10 追加）

### 新增字段
SealInfo 新增以下字段，从 PDF 签名字典中提取：
- signMethod: 签名方法 (/Filter + /SubFilter, 如 "Adobe.PPKMS / adbe.pkcs7.sha1")
- certIssuer: 证书颁发者 (从签名字典/CMS 证书)
- certSubject: 证书主题 DN
- certStartTime / certEndTime: 证书有效期
- certAlgorithm: 证书算法标识
- certData: 证书原始数据 (base64)

### 实现
- types.rs: 新增 7 个 Option<String> 字段
- engine.rs: parse_existing_signatures 从 /V 签名 dict 提取 /Filter, /SubFilter, /Name, /M 等
- index.html: 属性弹框全部显示，验证弹框补充部分字段
