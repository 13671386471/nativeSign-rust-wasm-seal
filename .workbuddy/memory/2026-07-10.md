## 1. 修复 ToUnicode 去重导致 Kangxi 字根泄漏 (font_embed.rs)
- 问题: 嵌入产物文本提取出现 ⼀/⼄/⼯ (Kangxi, U+2F00~) 而非 一/乙/工。

- 修复: 新增 `unicode_priority()`, 排序时按标准度优先再 dedup; 重建 pkg_node + pkg(web, release)。
- 验证(node 真 wasm + pypdf): Kangxi/兼容区/字根补充=0, 原文/嵌入 389 去重 CJK 完全一致, tofu=0。

## 6. 增强签章属性字段提取（2026-07-10 追加）

### 新增字段
SealInfo 新增以下字段，从 PDF 签名字典中提取：
- signMethod: 签名方法 (/Filter + /SubFilter, 如 "Adobe.PPKMS / adbe.pkcs7.sha1")
- certIssuer: 证书颁发者 (从签名字典/CMS 证书)
- certSubject: 证书主题 DN
- certStartTime / certEndTime: 证书有效期
- certAlgorithm: 证书算法标识
- certData: 证书原始数据 (base64)

### 实现
- types.rs: 新增 7 个 Option<String> 字段
- engine.rs: parse_existing_signatures 从 /V 签名 dict 提取 /Filter, /SubFilter, /Name, /M 等
- index.html: 属性弹框全部显示，验证弹框补充部分字段

## 7. SES 签章嵌入实现 (GM/T 0031 + PKCS#7 + GB/T 33190)

### 新增模块
- `src/der.rs`: 最小 ASN.1 DER 编码器 (SEQUENCE/SET/INTEGER/OID/BIT STRING/OCTET STRING/NULL/UTCTime 等)
- `src/ses.rs`: SES 电子印章/签名结构 (GM/T 0031-2014), MOCK 证书数据
- `src/pkcs7.rs`: PKCS#7/CMS SignedData 封装, 含 X.509 证书 Issuer/Serial 提取
- `src/ofd_sign.rs`: GB/T 33190 OFD 签章嵌入 (Sign_N.xml, Seal.esl, SignedValue.dat)
- `src/mock_seal_png.rs`: 120x120 红色圆形印章 PNG 字节数组
- `mock_data/`: mock_rsa_cert.der, mock_rsa_key.der, mock_ec_cert.der, mock_ec_key.der, mock_seal.png
- `scripts/gen_mock_cert.py`: 用 Python cryptography 生成 RSA(2048) + EC(P-256) 自签名证书

### 改写模块
- `src/seal.rs`: PDF 增量更新嵌入签章 — 创建 Sig Widget/Annot, Form XObject 外观流, Image XObject,
  计算 ByteRange, 填充 PKCS#7 到 /Contents; `finalize_pdf_signatures()` 两遍处理
- `src/sign.rs`: 集成 SES 签名流程 — prepare_for_signing/get_value_ex/set_value_ex
- `src/lib.rs`: 新增模块声明 + 5 个 WASM 导出函数 (build_ses_seal, build_ses_signature,
  build_pkcs7_signature, embed_signatures, get_ses_info)

### 验证结果 (tools/test_ses_struct.cjs, Node 22 + pkg_node 真 wasm)
- SES_Seal (SM2): 1344 bytes, 外层 SEQUENCE ✓, eSealInfo/signatureAlgo/signature 结构 ✓
- SES_Signature (SM2): 655 bytes, SEQUENCE ✓
- PKCS#7 SignedData (SM2): 1555 bytes, ContentInfo→SignedData→version=1→digestAlgorithms→
  encapContentInfo→certificates→signerInfos 全部 ✓
- contentType OID: 1.2.840.113549.1.7.2 (signedData) ✓
- RSA 兼容: SES_Seal 1536 bytes, PKCS#7 2339 bytes ✓

### 技术要点
- lopdf 0.34 ObjectId 为 (u32, u16) 元组; Object::String 需匹配 (String, StringFormat) 元组
- SIGNATURE_CONTENTS_SIZE = 16384 bytes 占位; 两遍处理先填占位再计算真实 ByteRange
- SES_Seal = SEQUENCE(eSealInfo, signatureAlgo, signature); eSealInfo 含印章类型/名称/证书/图片
- PKCS#7 SignedData 版本=1 (RFC 5652), certificates 用 [0] IMPLICIT 包装
- OFD 签章: ZIP 内创建 Doc_0/Signs/Sign_N.xml + Sign_N/Seal.esl + Sign_N/SignedValue.dat

### 完整签章流程
1. 打开 PDF/OFD → 配置自动应用 (算法/格式)
2. 点"签章" → 弹出印章选择框 → 上传图片/用默认 → 输入签署人 → 确认
3. 在文档上点击落章位置 → `SetCurrentSealInfo` + `AddSeal` → 印章添加
4. 点"保存" → `save_to` 自动构建 SES_Signature + PKCS#7 → 增量更新嵌入 → 下载文件
5. 或点"嵌入签名" → `embed_signatures` 嵌入但不下载
